🟢 Sprite Activation App

App Documentation

A tablet-based brand activation game for 3-day retail events.
Players pick a Sprite can, answer a trivia question, and win a prize.

📋
Section 01

App Overview

3 Event Days
30 Bottles
2000 Stress Balls

A full-screen tablet app deployed at modern trade / retail activations. Players engage with the Sprite brand through a trivia quiz and are rewarded with a physical prize. All prize distribution is handled automatically — no staff input needed.

Technology: React + Vite, CSS Modules, localStorage for persistent data. Self-hosted MontserratZ font (all 9 weights) for fully offline use at events.

🔄
Section 02

Screen Flow

1
Home Screen
Player sees two overlapping Sprite cans — Zero Sugar (left) and Regular (right). Tapping a can triggers a glow effect and a 700ms delay before advancing. The burger menu (top-right) opens the staff dashboard.
2
Quiz Screen
A Greek trivia question appears inside a branded card frame. Three answer buttons are shown. Tapping any answer triggers a scale animation and an 800ms delay, then proceeds to the result — both correct and wrong answers advance to the win screen (all players win a prize).
3
Win Screen
Shows "ΣΥΓΧΑΡΗΤΗΡΙΑ ΚΕΡΔΙΣΕΣ" with a spring-animated gift image and prize name. The prize type (bottle or stress ball) is determined automatically by the prize distribution logic. A return button sends the player back to the Home Screen.
🎁
Section 03

Prize Distribution System

The prize system uses a global play counter stored in localStorage. Every play increments the counter. Every 66th play triggers a bottle prize. All other plays receive a stress ball. There are no daily limits — the counter runs continuously across all 3 event days.

totalGames % 66 === 0 → Sprite Bottle 🍶
(games 66, 132, 198, 264, 330 … up to 1980)

totalGames % 66 !== 0 → Stress Ball 🟢
(all other plays)
🍶
Sprite Bottle
Given every 66 plays · 30 bottles total · stops after bottle #30
1 per 66 plays
🟢
Antistress Ball
Given on every other play · ~2000 available for the event
All other plays

Example sequence: Plays 1–65 → stress ball  ·  Play 66 → bottle  ·  Plays 67–131 → stress ball  ·  Play 132 → bottle  ·  and so on.

After all 30 bottles have been distributed (play 1980), the remaining plays continue to receive stress balls. The counter never resets unless the staff manually resets via the dashboard.

Section 04

Staff Dashboard

Accessible via the hamburger menu (☰) on the Home Screen — top-right corner. Opens immediately with no PIN required.

Global totals — Total Plays, Bottles Given, Stress Balls Given
"Next bottle in X plays" countdown with large highlighted number
Per-day breakdown — each calendar day shows its own play count and prize totals with fill-bar progress indicators
Reset button — clears all localStorage data after a two-step confirmation (requires tapping "OK, Reset" to confirm)
💾
Section 05

Data Storage

All data is stored in the browser's localStorage under the key spriteEventData. This means data persists across page refreshes and browser restarts on the same device, but is isolated to that tablet. No server or internet connection is required.

Data tracked: global total plays · global bottles given · global stress balls given · per-day records (date, plays, bottles, stress balls).
